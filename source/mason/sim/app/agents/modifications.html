Modifications List

1/9/10 (J. S.)

1. Added scripting systematic analysis of parameters.  The simulation parameters file allows the user to systematically change parameter values after the key word:  SIMULATION_PARAMETERS, which is case sensitive.  The first line contains the parameter names that can be changed.  After the first word, which labels the time step, the parameter names must match those declared in AgentsSimulation.java.  The list of value in the rows that follow are the parameter values for simulations 1 to n.

2. Added automatic PNG saves.  If savePNGToFile = true, then a PNG of the window at  time t is saved at the printSteps intervals.  When scripted, PNGs are saved in the simulation data folder in folders named "run" + the run number (e.g., run1, run2, etc.).  PNG files are named "name" + simulation number "-" + time step (e.g., pic2-1000.png).

3. Data is collected by the Data Class.  Note that the simulation determines the number of columns data by the length of fileheaders String variable:

public String[] fileheaders ={"trial", "N", "Coop","Defect","WCoop","WDefect","simulations"};

So, to add a new column of data, add a new column string:

public String[] fileheaders ={"trial", "N", "Coop","Defect","WCoop","WDefect","simulations", "newData"};

Then get the newData and add it to the data bag (e.g., data.add(newData)).
